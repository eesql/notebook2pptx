# notebook2pptx
针对notebooklm等ai工具生成的报告pdf，基于AI 多模态重排与矢量化闭环方案实现pdf无损转换为可编辑的pptx文件。

## 技术架构
前端： Unstructured.io (布局识别专家)

核心： GPT-4o / Gemini 1.5 Pro (作为布局决策者) + Cairo (矢量绘图引擎)

后端： python-pptx + 自定义字体映射引擎

## 执行逻辑（闭环流程）
多模态解析： 将 PDF 页面转为高分辨率图片交给 AI，同时提取 PDF 原生文本元数据。AI 负责判断：「这是一个带有圆角的阴影卡片，背景是 #F0F0F0 渐变」。

矢量重建 (SVG to PPTX)： * 不直接转存背景，而是用代码在 PPT 中「重绘」背景形状。

使用 python-pptx 的 shapes.add_shape 还原所有的矩形、圆角和线条，确保背景完全可拖拽、改色。

智能字体映射： * 提取 PDF 嵌入字体子集。

通过相似度算法自动匹配系统最接近的可编辑字体（如 "Inter" 匹配 "Arial"），并保留字间距参数。

文本流注入： 根据 AI 识别的语义块（标题、副标题、正文、页码），创建原生的 PPT 占位符（Placeholder），而不仅仅是浮动文本框。

## 迭代主要技术点：

**1.坐标系的高精度转换，**建立了 PDF 到 PPT 的坐标映射逻辑。

2.**文本块的智能聚类（避免断行**），引入 **DBSCAN 思想** 的行聚类算法，将同一行的单词合并，并提取字体颜色和大小。

3.**矢量背景的形状还原**

**4.字体乱码与不兼容：** 解决 PDF 嵌入字体（如 `ABCDE+Inter-Bold`）在 PPT 中无法识别的问题。

**5.段落逻辑重组：** 现在的方案是按「行」生成的，我们要升级为按「段落」生成，这样你在 PPT 里按回车或删减文字时，布局才不会乱。

**6.多栏布局支持：** 自动识别 NotebookLM 常用的双栏或多栏排版。

**7.表格逻辑隔离**：首先识别页面中的表格区域，记录其坐标。

**8.元素过滤重叠**：在处理文本和背景色块时，自动忽略位于表格区域内的元素，防止重复生成。

**9.原生表格映射**：将 PDF 的行高、列宽等比缩放到 PPT 坐标系。
